# 邻里帮 V6.1 AI 协作角色

## 1. 产品经理

任务：

- 确认项目边界。
- 防止产品重新变成复杂下单平台。
- 保证每个阶段都能独立演示。

验收：

- 每阶段有明确目标、非目标和验收标准。

## 2. 适老化体验设计

任务：

- 检查首页入口是否少而清楚。
- 检查字体、按钮、表单是否适合老人。
- 避免复杂二级分类。

验收：

- 老人端主流程不超过：首页 -> 提交页 -> 成功页。

## 3. 前端实现

任务：

- 使用 Streamlit 实现可运行网页。
- 保持页面简洁、稳定、可现场演示。
- 每个阶段代码单独保存。

验收：

- `streamlit run app.py` 可以直接运行。
- 不依赖旧代码目录。

## 4. 数据建模

任务：

- 用 SQLite 保存老人和工单。
- 图片存到本地 uploads 目录。
- 费用、后台、审核等后续阶段再扩展。

验收：

- 工单能保存并在“我的工单”里看到。

## 5. 测试审查

任务：

- 检查语法。
- 检查核心流程。
- 检查目录是否干净。

验收：

- 没有旧 quick_demo/web_demo/ppt_demo 代码干扰。
- Phase 1 目录能独立运行。
