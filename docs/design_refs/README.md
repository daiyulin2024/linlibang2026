## Phase 1 最新方向

- 首页采用“安靖社区服务台”方向。
- 中间功能卡片顺序：生活需求、上门事项、外出协助、社区反馈。
- 首页保留克制的语音入口，但不让语音按钮成为整页唯一主角。
- 首页不重复放“人工帮助”和“电话联系”两个入口，人工介入能力合并到服务台模块和提交页电话联系开关。
- 提交页采用“社区值班协助 + 清晰表单”的混合方向。
- 正式规格见：`../specs/08_phase_1_home_and_request_flow.md`

## 设计取向

- 简洁、高效、适老化，优先保证可读性和操作确定性。
- 保留 Apple 式克制感：浅背景、大字号、清晰层级、少装饰。
- 首页减少二级分类负担，入口直接对应老人能理解的需求类型。
- 提交页统一为“文字说明 + 语音留言 + 上传照片 + 紧急程度 + 是否需要社区电话联系”。

## 后续使用规则

- 后续 Phase 1 开发先按这两张图作为界面基准。
- 不继续生成新视觉方向，除非当前方向无法满足功能。
- 使用 `huashu-design` 做设计审查时，重点检查：通用 AI 风格、无意义装饰、按钮尺寸、字体层级、老人操作路径是否清楚。
